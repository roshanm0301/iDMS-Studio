# UI Studio Requirements

This folder contains product-specific requirements for each UI Studio milestone.

## How to use

Add one or more requirement documents per milestone before that milestone is executed.
Use the naming convention: `M<N>-<short-name>.md` (e.g. `M1-view-registry.md`).

The agent executing each milestone will read all requirement files in this folder
prefixed with the milestone number before starting implementation.

## What to include in a requirements document

- Business context and user stories for the feature
- Specific entity/field names to use (beyond the generic demo data)
- UI wireframes or descriptions
- Any domain-specific rules or constraints
- Priority ordering of sub-features within the milestone
- Explicit acceptance criteria beyond the methodology's exit criteria
